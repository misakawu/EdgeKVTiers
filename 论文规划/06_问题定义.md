## 5.2 系统模型

> 必须配系统模型图：请求流 → 命中检查 → (命中/恢复/重算) → COP 更新画像 → 显存压力下 LPE 决定准入/TTL/驱逐/offload → 复用时 RRS 选恢复或重算。

### 5.2.1 系统组成与交互

| 实体 | 符号 | 角色 |
| --- | --- | --- |
| 缓存对象 | $o\in\mathcal{O}$ | 有生命周期的运行时资产 |
| 请求 / 会话流 | $\mathcal{R}$ | 决定复用模式与显存压力 |
| GPU 显存池 | $M_{budget}$ | 受限资源 |
| offload 通道 | $BW,d_{deser}$ | 恢复代价来源 |

**交互描述**：请求到达时先查缓存；命中则省去对应前缀的 prefill；未命中（曾被驱逐到 offload）则由 RRS 决定恢复还是重算。COP 持续为每个对象维护 $p_{reuse}/c_{restore}/c_{recomp}/risk_{exp}$ 画像。显存压力上升时 LPE 依画像决定准入、TTL、驱逐去向（drop/offload/pin）。本题做**策略层**（在 vLLM/LMCache 外围），不重写底层 paged kernel。

### 5.2.2 代价模型

**单请求 TTFT**：设请求前缀含已缓存部分与未缓存部分，

$$TTFT(r) = \underbrace{c_{re}\cdot n_{miss}}_{\text{未缓存部分重算}} + \underbrace{\mathbb{1}[\text{复用且被驱逐}]\cdot c_{act}(o)}_{\text{恢复或重算缓存部分}}$$

其中复用时缓存部分的代价

$$c_{act}(o)=\min\big(c_{restore}(o),\ c_{recomp}(o)\big),\quad c_{restore}(o)=\frac{size(o)}{BW}+d_{deser},\quad c_{recomp}(o)=c_{re}\cdot n(o)$$

**显存约束**：$\sum_{o:\,resident} size(o)\le M_{budget}-M_{weights}-M_{buffer}$。

### 5.2.3 系统假设

| # | 假设 | 合理性 | 不成立的影响 | 实验是否验证 |
| --- | --- | --- | --- | --- |
| A1 | 重算代价线性于 token，$c_{recomp}=c_{re}\cdot n$ | prefill 近线性 | 强非线性需分段 | ✅ E1 拟合 $c_{re}$ |
| A2 | offload 带宽 $BW$ 稳定可测 | PCIe/NVMe 实测波动有限 | 波动大需在线估计 | ✅ E4 扫不同 $BW$ |
| A3 | 缓存复用**不改变答案**（同前缀同结果）| prefix/RAG chunk 语义等价 | 近似复用需另评质量 | ✅ 正确性回归 |
| A4 | 策略层不改底层 kernel | 接 LMCache/vLLM 接口 | 深度优化需改引擎 | ❌（边界）|

## 5.3 优化目标

EdgeKVLife 是一个**显存受限的在线缓存生命周期问题**：在显存预算内，决定每个对象的准入/TTL/驱逐与复用动作，以最小化 p95 TTFT。

$$\min_{adm,\ ttl,\ evict,\ act}\quad TTFT_{p95}(\mathcal{R}) + \lambda\cdot T_{policy}$$

subject to：

- **(C1) 显存约束**：$\sum_{o:\,resident} size(o)\le M_{budget}-M_{weights}-M_{buffer}$ —— 来源：端侧 GPU 显存。
- **(C2) 正确性约束**：缓存复用不得改变答案（命中即等价复用）—— 来源：prefix/chunk 语义等价。
- **(C3) I/O 不反噬**：$\mathbb{E}[c_{act}(o)]\le c_{recomp}(o)$，即恢复不比重算更慢 —— 来源：RRS（见定理 1）。
- **(C4) 策略开销**：$T_{policy}$ 远小于命中省下的 prefill —— 来源：策略层可行性。

> **目标含义**：可以为降 TTFT 而缓存/offload，但不能超显存、不能改答案、不能让 offload 的 I/O 比重算还贵、不能让策略本身吃掉收益。

## 5.4 核心定义与定理

**定义 1 — 缓存收益**：命中对象 $o$ 省下的 prefill = $c_{recomp}(o)=c_{re}\cdot n(o)$。

**定义 2 — 对象保留得分**（LPE 驱逐依据）：$score(o)=\dfrac{p_{reuse}(o)\cdot c_{recomp}(o)}{size(o)}$，即"单位显存的期望节省"。

**定义 3 — 临界带宽**：$BW^\* = \dfrac{\mu_{kv}}{c_{re}}$（每 token KV 大小与每 token 重算代价之比）。

**定理 1（restore-vs-recompute 阈值）**：对被驱逐到 offload 的对象 $o$（$n$ 个 token），当再次复用时，**恢复比重算更划算当且仅当**

$$\frac{\mu_{kv}\cdot n}{BW} + d_{deser} \le c_{re}\cdot n \iff BW \ge \frac{\mu_{kv}\cdot n}{c_{re}\cdot n - d_{deser}} \xrightarrow{n\ \text{大}} BW^\*=\frac{\mu_{kv}}{c_{re}}.$$

**证明**：直接比较 $c_{restore}(o)=size(o)/BW+d_{deser}=\mu_{kv}n/BW+d_{deser}$ 与 $c_{recomp}(o)=c_{re}n$。令前者 $\le$ 后者并解出 $BW$ 即得；当 $n$ 足够大、$d_{deser}$ 相对可忽略时阈值收敛到 $BW^\*=\mu_{kv}/c_{re}$。$\square$

> **物理含义**：存在一个临界带宽 $BW^\*$——offload 通道快于它则恢复总更划算，慢于它则重算更划算。RRS 据此在线选择动作，避免"盲目 offload 反被 I/O 拖慢"。

**注（竞争比）**：LPE 的驱逐是带权缓存（weighted caching）的在线问题；离线最优为 Belady。按 $score(o)$ 的 GreedyDual 式策略在带权缓存上有已知竞争比上界（$O(\log k)$ 级，$k$ 为缓存容量）。本题不重证该上界，而以它说明"$score$ 驱逐"有理论支撑、并用 E2 的 trace 实测逼近 Belady 的差距。

## 5.5 与 baseline 的问题定义差异

| baseline | 它怎么定义问题 | EdgeKVLife 新增了什么 |
| --- | --- | --- |
| LRU / LFU | 按时近 / 频次驱逐 | 按 $score=p_{reuse}\cdot c_{recomp}/size$ 驱逐（含恢复代价）|
| vLLM/PagedAttention | 分页管理 KV 内存 | 在 workload 层决定生命周期（准入/TTL/offload）|
| SGLang/RadixAttention | prefix 复用 | 加入 offload 与 restore-vs-recompute (C3)/定理 1 |
| LMCache | 提供 cache layer | 提出可解释生命周期策略，而非再实现一层 cache |
