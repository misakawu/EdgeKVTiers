# 十一章 — 检查清单
---

## 11.1 阶段验收表

| 阶段 | 验收材料 | 通过标准 | 导师意见 |
| --- | --- | --- | --- |
| 文献阶段 | 最强基线卡片、SOTA 表、Related Work 分类。 | 能讲清每个强基线的方法、局限和本题差异。 |  |
| 预实验阶段 | E1 报告（命中收益、$c_{re}$）、Go/No-Go。 | 命中省 prefill 可测，vLLM 默认可复现。 |  |
| 方法阶段 | 方法图、COP/LPE/RRS 伪代码、定理 1 证明、§6.5 trace。 | 每模块有输入/输出/日志，RRS 阈值与定理一致。 |  |
| 实验阶段 | 主实验、消融、参数、端侧和失败案例。 | 结果能支撑贡献点，负结果能解释。 |  |
| 写作阶段 | 论文 v0.9、附录、复现说明。 | 图表、符号、实验和结论一致。 |  |

## 11.2 投稿前硬门槛

| 门槛 | 最低要求 | 未达到时的处理 |
| --- | --- | --- |
| 强基线 | 不少于三个可解释的强 baseline 或等价实现。 | 补 baseline，不进入投稿。 |
| 质量约束 | 所有系统收益都同时报告质量变化。 | 补质量指标和失败案例。 |
| 端侧证据 | 至少一组低资源或端侧趋势验证。 | 降级为限制讨论或补实验。 |
| 策略开销 | policy overhead 单独报告。 | 重新拆分计时。 |
| 复现材料 | 配置、随机种子、结果文件可追溯。 | 整理实验包后再投。 |

## 11.3 自查问题清单（每阶段结束自问）

- 文献：我能否对 §0.3 任一篇说清"优化对象、决策变量、是否报系统收益"？我落在哪个空白（对象级生命周期 + 边缘小显存 + restore-vs-recompute）？
- 预实验：H1/H2/H3 哪几个通过？LPE 比 LRU 降 p95 多少？RRS 在低带宽是否避免了 I/O 反噬？
- 方法：COP/LPE/RRS 能否各自关掉做消融？定理 1 的 $BW^\*$ 是否被 E4 实测验证？trace 是否可复盘每次决策？
- 实验：是否给了 Belady 上界对照？是否扫了带宽并报 recompute ratio？$T_{policy}$ 单独报告了吗？
- 落地：端侧收益是否真在受限 GPU/Jetson 测得？策略层不改引擎是否成立？
- 写作：一句话能否讲清"现有方法缺什么、我补什么"，且对应 §0.3 具体论文（如 vLLM 只管页、不管对象生命周期）？

## 11.4 检查清单使用方式

这张检查表不是最后一周才填写的材料。每次组会后都要更新一个阶段状态，并把证据文件、实验日志或论文图表补齐。如果某个阶段连续两次无法推进，应进入风险复盘，而不是继续写“进行中”。
