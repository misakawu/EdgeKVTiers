# pre — 纯 Python Trace 预仿真

## 1. 实验定位

本实验对应 [00_预实验提取.md](/D:/EdgeKVLife/论文规划/00_预实验提取.md:1) 中的 `0.5.0-pre`。它不是系统部署实验，而是用纯 Python 和 trace 数据快速模拟 `H1/H2/H4/H5`，判断自变量变化是否会稳定影响因变量。

`pre` 的结论只回答“方向是否值得继续”，不作为最终算法有效性证据。算法有效性证据放到 `H0` 提供的服务器端/边缘端无 `LMCache` 模拟器中完成；可部署性证据放到 `H3` 的 `LMCache` 接入实验中完成。

## 2. 回答什么 / 卡住后续什么

本实验要回答四个趋势问题：

1. `H1` 趋势：显存预算 `M_budget` 变化时，`score/LPE` 是否相对 `LRU/LFU` 降低 `p95 TTFT`。
2. `H2` 趋势：带宽 `BW` 变化时，`RRS` 是否贴近 `always-restore` 与 `always-recompute` 的较优下包络。
3. ~~`H4` 趋势：质量预算 `epsilon` 变化时，`TMS-tiered` 是否相对二元 cache 或 `LPE-only` 有额外收益。~~ 修改方案：`H4` 继续在内部使用绝对预算 `epsilon_abs` 执行，但文档、图表与结果汇总统一追加归一化质量预算 `epsilon_norm = epsilon_abs / token_ref`，主趋势描述改为“`epsilon_norm` 变化时 `TMS-tiered` 是否相对二元 cache 或 `LPE-only` 有额外收益”。 
4. ~~`H5` 趋势：`(BW, epsilon)` 网格变化时，最优精度 `q*` 是否呈现可解释分区。~~ 修改方案：`H5` 内部仍按 `epsilon_abs` 跑网格，但热力图、结果表和后续文档统一使用 `(BW, epsilon_norm)` 网格表达，并同步记录 `token_ref` 与 `epsilon_abs`。

若 `pre` 看不到任何趋势，先调整 trace、参数范围和示意三元组，不进入 `H0/H3`。

## 3. 前置依赖

1. 不依赖真实模型、GPU、`vLLM` 或 `LMCache`。
2. 需要一条 trace，可由 ShareGPT/RAG 请求转成对象访问序列，也可先人工构造。
3. 需要一组示意代价参数：`mu_kv`、`c_re`、`d_deser`、各精度 `(size_factor, quality_loss, restore_factor)`。

## 4. 使用环境与资源

1. 运行环境：本地 Python。
2. 数据与负载：`ShareGPT` 多轮前缀复用 trace、RAG chunk 复用 trace，或人工构造的最小复用序列。
3. 对象字段：`object_id`、`n_tokens`、`reuse_key`、`arrival_ts`、`object_type`、`p_reuse`。
4. 策略集合：`LRU`、`LFU`、`score/LPE`、`TMS-tiered`、`always-restore`、`always-recompute`、`RRS`。
5. 精度等级：`full`、`int8`、`int4`、`sparse_k`、`offload/drop`。
6. ~~参数网格：`M_budget` 三档、`epsilon` 三档、`BW` 多档。~~ 修改方案：参数网格保留 `M_budget` 三档与 `BW` 多档，质量预算改为“内部执行 `epsilon_abs` 三档、输出与图表展示 `epsilon_norm` 三档”，并为每次运行记录 `token_ref` 以完成绝对值到归一化值的换算。

## 5. 输入与记录字段

1. 输入字段：`request_id`、`object_id`、`n_tokens`、`arrival_ts`、`reuse_key`
2. ~~自变量字段：`policy`、`M_budget`、`BW`、`epsilon`~~ 修改方案：自变量字段扩展为 `policy`、`M_budget`、`BW`、`epsilon_abs`、`epsilon_norm`、`token_ref`，其中内部执行读取 `epsilon_abs`，后处理与图表优先使用 `epsilon_norm`。
3. 决策字段：`q_before`、`q_after`、`lpe_action`、`rrs_action`、`tms_action`
4. ~~因变量字段：`ttft_ms`、`hit`、`size_mb`、`qloss_delta`、`qloss_total`~~ 修改方案：因变量字段扩展为 `ttft_ms`、`hit`、`size_mb`、`qloss_delta_abs`、`qloss_total_abs`、`qloss_delta_norm`、`qloss_total_norm`，并保留绝对值与归一化值双口径。

## 6. 实验执行步骤

1. 编写纯 Python `sim.py`，实现 trace 回放和对象状态机。
2. 跑 `H1` 趋势：扫 `M_budget`，比较 `LRU/LFU/score` 的 `p95 TTFT`、命中率和显存峰值。
3. 跑 `H2` 趋势：扫 `BW`，比较 `always-restore/always-recompute/RRS`。
4. ~~跑 `H4` 趋势：扫 `(M_budget, epsilon)`，比较二元 cache、`LPE-only`、`static-int4`、`TMS-tiered`。~~ 修改方案：跑 `H4` 时内部扫 `(M_budget, epsilon_abs)`，但输出矩阵和图表按 `(M_budget, epsilon_norm)` 汇总，并同时给出每个 `epsilon_norm` 对应的 `epsilon_abs`。
5. ~~跑 `H5` 趋势：扫 `(BW, epsilon)`，记录每个网格点的 `q*`。~~ 修改方案：跑 `H5` 时内部扫 `(BW, epsilon_abs)`，热力图与 CSV 统一使用 `(BW, epsilon_norm)` 网格记录 `q*`，同时输出对象集的 `token_ref`。
6. 将每次运行的配置和结果落到 CSV/JSONL，作为 `H0` 模拟器参数范围的输入。

## 6.1 对应伪代码

```python
MU_KV, C_RE, D_DESER = 0.12, 0.12, 3.0
TIERS = {
    "full": (1.00, 0.000, 1.00),
    "int8": (0.50, 0.002, 1.00),
    "int4": (0.25, 0.008, 1.05),
    "sparse_k": (0.20, 0.015, 1.10),
}

def size(o, q):
    return MU_KV * o.n * TIERS[q][0]

def qloss(o, q):
    return TIERS[q][1] * o.n

def c_recomp(o, q):
    return C_RE * o.n * TIERS[q][2]

def c_restore(o, q, bw):
    return size(o, q) / bw + D_DESER

def score(o, q):
    return o.p_reuse * c_recomp(o, q) / size(o, q)

def rrs(o, q, bw):
    return "restore" if c_restore(o, q, bw) <= c_recomp(o, q) else "recompute"
```

## 7. 产出物

1. 纯 Python 仿真脚本。
2. `H1/H2/H4/H5` 趋势图和趋势表。
3. ~~参数范围建议：`M_budget`、`BW`、`epsilon`、精度三元组。~~ 修改方案：参数范围建议改为 `M_budget`、`BW`、`epsilon_norm`、`epsilon_abs` 映射表、精度三元组；后续 H0/H4/H5 统一以 `epsilon_norm` 作为对外口径。
4. `H0` 模拟器需要支持的 trace 字段和指标字段清单。

## 8. 通过标准

1. `H1`：紧显存档下 `score/LPE` 相比 `LRU/LFU` 出现稳定优势。
2. `H2`：`RRS` 在多数带宽档不劣于两种固定策略中的较优者。
3. ~~`H4`：`TMS-tiered` 在同 `epsilon` 下优于二元 cache 或 `LPE-only`。~~ 修改方案：`H4` 的通过表述改为“在同 `epsilon_norm` 下，`TMS-tiered` 对应的 `QLoss_norm` 受控且优于二元 cache 或 `LPE-only`”，内部仍通过 `qloss_abs <= epsilon_abs` 实现。
4. ~~`H5`：`(BW, epsilon)` 网格上能看到非随机的 `q*` 分区。~~ 修改方案：`H5` 的通过表述改为“`(BW, epsilon_norm)` 网格上能看到非随机的 `q*` 分区，并可回溯到对应的 `epsilon_abs` 与 `token_ref`”。

## 9. 与后续实验衔接
