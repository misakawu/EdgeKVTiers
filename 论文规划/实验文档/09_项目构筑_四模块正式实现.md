# 项目构筑 — 四模块正式实现

## 1. 构筑定位

本构筑项对应 [实验方案.md](/D:/EdgeKVLife/论文规划/实验方案.md:189) 的 `2.1`，承接 [00_王祯祥_论文工作规划.md](/D:/EdgeKVLife/论文规划/00_王祯祥_论文工作规划.md:1329) 到 [1466](/D:/EdgeKVLife/论文规划/00_王祯祥_论文工作规划.md:1466) 的四模块方法设计。

## 2. 目标

将仿真器中已验证的逻辑固化为真实工程模块，并保证每个模块可单独开关、可单独记日志、可单独消融。

## 3. 实现范围

1. `COP`：对象画像更新
2. `TMS`：在线精度选择与迁移控制
3. `LPE`：驱逐、TTL、offload/drop 判决
4. `RRS`：复用时 restore/recompute 判决

## 3.1 构筑所需环境与资产

1. 代码环境：Python、PyTorch、Transformers、vLLM、LMCache；版本与 CUDA/driver 信息沿用 `H0/H3` 的 `env.lock`。
2. 模型配置：默认面向 `Qwen2.5-7B-Instruct` 的 KV 尺寸、tokenizer 和上下文长度配置实现；接口需保留切换到 `Llama-3.1-8B-Instruct`、`DeepSeek-R1-Distill-Qwen-7B` 的能力。
3. 数据资产：使用 `H0` 统一 `trace.jsonl`、`H4` 的 `(alpha,beta,ell)` 三元组表、`H5` 对象集合与带宽/质量预算配置。
4. 机制层接口：`KIVI/KVQuant` 提供 `int8/int4` 量化路径，`H2O/SnapKV` 提供 `sparse_k` 路径，`LMCache` 提供 admission/eviction/offload/reuse 钩子。
5. 运行硬件：服务器 GPU 用于主构筑验证，端侧或受限显存 GPU 用于检查模块开销和显存压力触发路径。
6. 日志规范：所有模块统一输出对象画像、精度迁移、驱逐、restore/recompute、质量损失、策略耗时和迁移耗时字段，供 `E2-E6` 直接复用。

## 4. 接口要求

1. `update(o, access_event)`
2. `on_pressure(resident, epsilon_rem)`
3. `evict_one(resident)`
4. `on_reuse(o, q, BW_est)`

## 5. 代码要求

1. 所有决策写入统一 JSONL trace
2. 所有模块通过配置开关独立启停
3. 所有参数来自统一配置文件，不允许散落在代码中

## 5.1 对应伪代码

### COP

```text
Algorithm 1: COP.update(o, access_event, q)
────────────────────────────────────────────────
1:  n(o)               ← token_count(o)
2:  for q in Q:
3:      size(o, q)     ← α(q) * μ_kv * n(o)
4:      c_recomp(o, q) ← β(q) * c_re * n(o)
5:      c_restore(o,q) ← size(o,q)/BW + d_deser
6:      ℓ(o, q)        ← measure_quality_loss(o, q)
7:  p_reuse(o)         ← estimate_reuse(o)
8:  risk_exp(o)        ← expiry_risk(o)
9:  score(o, q)        ← p_reuse(o) * c_recomp(o,q) / size(o,q)
10: return profile(o)
```

### TMS

```text
Algorithm 2: TMS.on_pressure(resident, profiles, M_budget, ε, BW)
────────────────────────────────────────────────────────────────
1:  ε_remaining ← ε - cumulative_qloss()
2:  while mem_used(resident) > M_budget - reserve:
3:      o* ← argmin_{o in resident, not pinned} score(o, q(o))
4:      q_cur ← q(o*); q_next ← downgrade_tier(q_cur)
5:      Δℓ ← ℓ(o*, q_next) - ℓ(o*, q_cur)
6:      if Δℓ ≤ ε_remaining and feasible(o*, q_next, BW):
7:          if hedge_choose(o*, q_next) == "accept":
8:              quantize_inplace(o*, q_cur → q_next)
9:              q(o*) ← q_next; ε_remaining -= Δℓ
10:             c_mig_total += c_mig(q_cur → q_next)
11:             N_mig += 1
12:             continue
13:     break
14: if mem_used(resident) > M_budget - reserve:
15:     call LPE.on_TMS_exhausted()
```

### LPE

```text
Algorithm 3: LPE.on_TMS_exhausted(resident, profiles, M_budget)
────────────────────────────────────────────────
1:  while mem_used(resident) > M_budget - reserve:
2:      o* ← argmin_{o in resident, not pinned, q(o)=lowest_tier} score(o, q(o))
3:      if p_reuse(o*) ≥ θ_keep and feasible_offload(o*):
4:          evict(o*) ← offload
5:      else:
6:          evict(o*) ← drop
7:      resident.remove(o*)
8:  on_admit(o_new): adm(o_new) ← (score(o_new, full) > min_score(resident))
9:  ttl(o) ← f(risk_exp(o))
```

### RRS

```text
Algorithm 4: RRS.on_reuse(o, q)
────────────────────────────────────────────────
1:  if expired(o):  return skip
2:  c_restore ← α(q) * μ_kv * n(o) / BW + d_deser
3:  c_recomp  ← β(q) * c_re * n(o)
4:  if c_restore ≤ c_recomp:
5:      return restore
6:  else:
7:      return recompute
```
