# H3 — 接入 LMCache 的可部署性验证

## 1. 实验定位

`H3` 是预实验阶段唯一接入 `LMCache` 的实验。它不重复 `H1/H2/H4/H5` 的算法有效性验证，而是检查已经在 `H0` 模拟器中通过的 `COP/TMS/LPE/RRS` 能否挂到 `LMCache/vLLM` 的真实 cache layer 上，并为主实验提供可运行环境。

## 2. 回答什么 / 卡住后续什么

本实验要回答：

1. `COP/TMS/LPE/RRS` 是否能通过 `LMCache` 或等价 hook 接入真实 cache 生命周期。
2. 接入后 trace 字段、策略动作和指标是否与 `H0` 模拟器保持语义一致。
3. 策略层开销 `T_policy`、迁移开销 `c_mig` 是否小于收益，且不破坏 serving 稳定性。
4. 服务器端和边缘端是否都能跑通最小可部署链路。

若 `H3` 不通过，主实验不能直接宣称系统可落地，只能退回 `H0` 模拟器证据或缩小主实验范围。

## 3. 前置依赖

1. `H0` 已提供稳定的无 `LMCache` 模拟器。
2. `H1/H2/H4/H5` 已在 `H0` 中得到算法有效性证据。
3. 已确认 `LMCache/vLLM` 或等价 cache layer 的 admission、eviction、reuse、offload/restore 接入点。
4. 已准备与 `H0` 相同结构的 trace 和配置。

## 4. 使用环境与资源

1. 服务器环境：1 张服务器 GPU，运行 `vLLM + LMCache` 或等价 cache layer。
2. 边缘端环境：Jetson/边缘 GPU，或服务器 GPU 的限显存模拟环境。
3. 模型：优先使用 `Qwen2.5-7B-Instruct` 或端侧可承载的同系列小模型/4-bit 版本。
4. 数据与负载：与 `H0` 相同结构的 `ShareGPT + RAG` trace；H3 可缩小规模，但字段必须一致。
5. 策略接入：先接 `COP + LPE + RRS`，稳定后再接 `TMS` 的降级路径。
6. 对照配置：`LMCache default`、`LMCache + LPE/RRS`、`LMCache + COP/TMS/LPE/RRS`。

## 5. 输入与记录字段

1. hook 字段：`on_admit`、`on_evict`、`on_reuse`、`on_pressure`
2. 策略字段：`lpe_action`、`rrs_action`、`tms_action`、`q_before`、`q_after`
3. 开销字段：`T_policy`、`c_mig`、`restore_latency`
4. 系统字段：`TTFT_p95`、`hit_rate`、`M_peak`、`error_count`
5. 一致性字段：`h0_event_id`、`lmcache_event_id`、`semantic_match`

## 6. 实验执行步骤

1. 用同一 trace 跑 `LMCache default`，获得可部署 baseline。
2. 确认每类 hook 的触发时机，并写最小事件日志。
3. 接入 `COP`，确保对象画像字段与 `H0` 语义一致。
4. 接入 `LPE` 的 eviction 决策和 `RRS` 的 restore/recompute 决策。
5. 接入 `TMS` 降级路径，并单独记录 `c_mig`。
6. 计时每次 hook 内 `T_policy`，记录 p50/p95。
7. 服务器端跑完整 smoke test，边缘端跑缩小 trace。
8. 对比 `H0` 与 `H3` 的事件语义，列出不一致样本。

## 6.1 对应伪代码

```python
cache = attach_lmcache(engine)
policy = EdgeKVTiersPolicy(config)

def timed_hook(name, fn, *args):
    t0 = time.perf_counter()
    action = fn(*args)
    t_policy_ms = (time.perf_counter() - t0) * 1000
    log_event(hook=name, action=action, t_policy_ms=t_policy_ms)
    return action

cache.on_admit = lambda obj: timed_hook("admit", policy.cop_update, obj)
cache.on_reuse = lambda obj: timed_hook("reuse", policy.rrs_on_reuse, obj)
cache.on_pressure = lambda resident: timed_hook("pressure", policy.tms_then_lpe, resident)
cache.on_evict = lambda resident: timed_hook("evict", policy.lpe_choose_victim, resident)
```

## 7. 产出物

1. `LMCache + EdgeKVTiers` 最小可运行原型。
2. hook 触发和策略动作 JSONL。
3. `T_policy/c_mig` 开销拆解表。
4. 服务器端与边缘端 smoke test 报告。
5. `H0` 模拟器事件与 `H3` 真实 hook 事件的一致性检查表。

## 8. 通过标准

1. 四个模块至少能以最小功能接入 `LMCache` hook 链路。
2. 同一 trace 可跑完，错误率可控，事件日志完整。
3. `T_policy` p95 小于 1 ms，且明显小于单次 cache 命中或避免重算带来的收益。
4. 服务器端和边缘端均能跑通 smoke test。
5. 与 `H0` 的关键事件语义一致，不一致项有明确解释。

## 9. 与后续实验衔接

`H3` 通过后，主实验 `E2` 可以使用 `LMCache + EdgeKVTiers` 环境。若只通过服务器端而边缘端失败，主实验可先保留服务器结果，端侧验证降级为后续风险项；若 `LMCache` hook 无法支撑策略动作，应回到项目构筑阶段调整接口。

## 10. 当前代码适配进展（2026-06-10）

### 10.1 代码落点

1. `h0/h3_lmcache_adapter.py`
   - 新增 `EdgeKVTiersPolicy`，作为挂到 `LMCache/vLLM` cache layer 的策略层对象。
   - 新增 `attach_to_cache(cache, policy)`，支持两类接入方式：
     - `cache.register_hook(name, fn)`
     - `cache.on_admit/on_reuse/on_pressure/on_evict` 属性赋值
   - 不直接 import `LMCache` 或 `vLLM`，避免把 H0/H1/H2/H4/H5 的仿真环境和真实 serving 环境强绑定。

2. `h0/run_h1245.py`
   - 新增 `--h3-contract`，用于输出 H3 接入契约。
   - 新增 `--emit-h3-events`，用于把 H0 事件转换为 H3 hook 日志样例。
   - 新增 `--h3-event-limit`，控制样例事件数量。

### 10.2 H3 hook 与四模块映射

1. `on_admit -> COP`
   - 入口函数：`EdgeKVTiersPolicy.cop_update`
   - 记录对象 admission、对象 id、当前 tier。

2. `on_reuse -> RRS`
   - 入口函数：`EdgeKVTiersPolicy.rrs_on_reuse`
   - 根据 `c_restore <= c_recomp` 判定 `restore` 或 `recompute`。
   - 记录 `rrs_action` 和 `restore_latency_ms`。

3. `on_pressure -> TMS`
   - 入口函数：`EdgeKVTiersPolicy.tms_then_lpe`
   - 按 H0 中的 keep score 选择候选对象并尝试降级。
   - 记录 `tms_action`、`q_before`、`q_after`、`c_mig_ms`。

4. `on_evict -> LPE`
   - 入口函数：`EdgeKVTiersPolicy.lpe_choose_victim`
   - 在 TMS 无法继续释放内存时选择 offload/drop。
   - 记录 `lpe_action` 与对象 tier 变化。

### 10.5 当前边界

1. 当前代码完成的是“策略层插件契约”和“hook 语义适配”，不依赖实际安装的 LMCache/vLLM。
2. 真机 H3 运行时，需要把 `attach_to_cache(cache, policy)` 接到实际 cache layer 暴露的 hook 对象上。
3. 如果 LMCache 当前版本 hook 名称不同，只需要在 `attach_to_cache` 处增加名称映射，不需要修改 H1/H2/H4/H5 实验逻辑。
4. `T_policy_ms` 已在策略回调内部计时；真实 `c_mig_ms` 需要接入 KIVI/H2O 或等价量化/稀疏化路径后替换当前估计值。
